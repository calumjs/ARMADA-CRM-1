# Pitfalls — traps that bite in ARMADA-CRM-1

Footguns, generated files, and environment gotchas. Read before editing; each entry
is acted on by `shipwright` during planning.

## Prisma

### Windows-arm64 has no Prisma Rust query-engine binary — use the JS driver adapter
- **heuristic:** Do **not** rely on Prisma's bundled Rust query engine in this repo/environment.
  Windows-arm64 ships no native query-engine binary and it fails to load with
  *"not a valid Win32 application"*. Use **Prisma 7 + the `@prisma/adapter-better-sqlite3`
  driver adapter** (engine-less / `driverAdapters`) instead — the JS adapter sidesteps the
  missing binary entirely. `src/lib/db.ts` constructs `new PrismaBetterSqlite3({ url })` and
  passes it as `adapter` to `PrismaClient`. Keep `DATABASE_URL` swappable (Postgres/libSQL later).
- **evidence:** PR #9 "Key decisions" + `prisma.config.ts` comment + `src/lib/db.ts`: the default
  query engine "fails to load with 'not a valid Win32 application' on this environment"; the
  driver-adapter approach built clean (lint/test/build all green). muster confirmed the approach
  "sound and correctly motivated".
- **confidence:** High
- **source:** PR #9 · issue #1 · 2026-06-07

### Do not edit `src/generated/` — it is generated and gitignored
- **heuristic:** Never hand-edit anything under `src/generated/` (the Prisma client output). It is
  gitignored and regenerated by `prisma generate` on `postinstall` and `build`. Change the schema
  and regenerate instead — a manual edit is lost.
- **evidence:** `prisma/schema.prisma` `generator client { output = "../src/generated/prisma" }`;
  `.gitignore` adds `src/generated/`; `package.json` runs `prisma generate` on `postinstall` and as
  the first half of `build`.
- **confidence:** High
- **source:** PR #9 · 2026-06-07

### A `@/generated/prisma/client` "broken import" is NOT broken after `prisma generate`
- **heuristic:** Before flagging/blocking on an unresolved import from the generated Prisma client
  (e.g. `@/generated/prisma/client`), **ground-truth it against a real build**. With Prisma 7,
  `prisma generate` emits `client.js` + `client.d.ts` under `src/generated/prisma`, so the
  `/client` subpath resolves and `prisma generate && next build` succeeds. The file is absent from a
  fresh checkout (gitignored) but present after install/generate — static inspection alone yields a
  false positive.
- **evidence:** PR #9 — the codex-rescue review lens raised this exact import as a **blocking**
  finding; it was **refuted** by building the branch in an isolated worktree (`prisma generate` v7.8.0
  emitted the entrypoints; lint clean, 6/6 tests, build green, type-check green).
  Trail: https://github.com/calumjs/ARMADA-CRM-1/pull/9 (inline note on `prisma/schema.prisma`).
- **confidence:** High

## Tooling / environment

### `format:check` failures on Windows are usually CRLF line-ending noise
- **heuristic:** If `npm run format:check` (Prettier) reports style issues on a Windows checkout but
  the actual content diff is empty, treat it as CRLF line-ending noise, not a real defect — don't
  block on it.
- **evidence:** PR #9 muster review nit — "`format:check` reports style issues that are CRLF
  line-ending noise on a Windows checkout (the actual Prettier content diff is empty)".
- **confidence:** Medium
- **source:** PR #9 · 2026-06-07
